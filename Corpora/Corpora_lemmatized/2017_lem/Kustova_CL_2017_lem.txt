сначала программа исправлять всё ошибки, кроме той, который принадлежать один из выбрать преподаватель тип и который впоследствии должный быть исправить студент. получиться предложение, в который использовать для создание тест на основа другой корпусов, использовать brat. 2.3. оценка преподаватель по окончание работа retm преподаватель получать некоторый пул вопросов; в зависимость от количество текстов, подать на вход, и выбрать тип ошибок, количество вопрос мочь доходить до несколько тысяч. поскольку вопрос быть сгенерировать автоматически, некоторый из они мочь оказаться неподходящий для реальный теста. на это этап преподаватель необходимо вручную просмотреть всё получиться вопрос и убрать из пул тот из них, который не подходить для теста. помимо фильтрация вопросов, преподаватель необходимо также приписать каждый из они уровень сложности: низкий, средний или высокий. это нужно для того, чтобы вопрос в предел один тест варьироваться по сложности: за каждый верный ответ следовать вопрос более высокий (ить равного, если самый высокий уже достигнут) уровень сложности, и наоборот при неверный ответе. количество балл за правильный ответ прямо пропорционально уровень сложность вопроса. 3. преимущество и недостаток retm тест по английский язык должный удовлетворять ряд требований. так, важно, чтобы проверочный тест содержать вопросы, который бы позволять студент отработать в тот число наиболее слабый языковой навыки. в то же время необходимо, чтобы тест соответствовать предполагать уровень он владение языком. тесты, создать при помощь retm, удовлетворять описать требованиям. поскольку источник для вопрос являться ошибки, допустить такой же студентами, можно с определённый доля уверенность утверждать, что этот ошибка являться типичными. иначе говоря, такой вопрос направить именно на отработка наиболее слабый языковой компетенция студентов. ещё один следствие из выбор источник для вопрос заключаться в том, что уровень язык в вопрос примерно соответствовать навык владение английский у студентов, проходить тест. на это направить и система выставление уровень сложность — тест как бы дополнительно подстраиваться под уровень студента, выставлять он соответствовать баллы. нельзя не отметить, однако, некоторый ограниченность применение retm. так, на данный момент инструмент позволять создавать лишь вопрос на исправление ошибок, где ответ представлять себя короткий строку, который студент должный самостоятельно ввести с клавиатуры. такой вопрос зачастую не учитывать весь возможный вариант ответа, что приводить к возможность несправедливый оценивание теста. кроме того, объёмный тесты, состоять исключительно из вопрос один типа, мочь быть утомительный для студентов. на данный момент в разработка находиться инструмент, который позволить на основа тот же источник создавать вопрос на выбор верный вариант из списка. важно также, что ни retm, ни разметка в realec пока не являться безупречными, а потому для составление финальный вариант тест всё равно требоваться усилие эксперта. тем не менее, этот процесс всё равно требовать маленький время и сил, чем создание тест с нуля. поэтому retm позиционироваться не как полноценный замена живой составитель тестов, но как инструмент, позволять облегчить они работу.
