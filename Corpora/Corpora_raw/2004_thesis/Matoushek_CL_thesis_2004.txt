МОДЕЛИРОВАНИЕ  ДИАЛОГОВ  НА  ОСНОВЕ  КОРПУСА  ЕСТЕСТВЕННЫХ  ДИАЛОГОВ
Построение компьютерной информационной диалоговой системы, которая общается с пользователем на естественном языке, требует выполнения эффективного процесса тестирования, с помощью которого разработка главных модулей диалоговой системы может быть частично автоматизирована.
В данном докладе рассматриваются некоторые проблемы, возникавшие в ходе конструирования двуязычной диалоговой системы и связанные с автоматическим созданием корпуса для тестирования и моделирования диалогов. Авторы анализируют опыт автоматического создания разных структурных моделей предложений/высказываний с помощью специального программного пакета с использованием метода квантитативного лингвистического анализа переписанных и аннотированных естественных диалогов. Программный пакет создает набор формул (структурных моделей предложений), состоящих из элементов специальной грамматики и описывающих синтаксическую структуру необходимых предложений/высказываний, и на основе этого строит обширный корпус с предложениями для тестирования.
В докладе предлагается метод формализации естественных диалогов, записанных в городских информационных центрах, с помощью которого можно определять внутреннюю структуру диалогов в виде псевдо-поточной диаграммы. Это представление включает ряд простых диалоговых шагов, причем каждый из них составляет маленькую часть диалога, которая описывается 2-3 предложениями. Прежде всего, следует пометить диалоговые акты, отнести их к соответствующей проблемной области, выбрать типовые высказывания пользователей, произвести компьютерное сравнение переписанных высказываний пользователей с выбранными предложениями данной проблемной области, и распознать диалоговый шаг. Все это возможно с помощью эффективного диалогового менеджмента, который должен осуществляться в городских информационных центрах, поддерживающих общение с пользователями на немецком и на чешском языках. Экспериментально созданный корпус с предложениями для тестирования, грамматики и диалоговые модели использовались для разработки и тестирования модулей двуязычной городской информационной системы в городах-побратимах Пльзень и Регенсбург.
