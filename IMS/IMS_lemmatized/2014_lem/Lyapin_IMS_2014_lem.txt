распределённый запрос". базовый он являться функционально, то есть для это конкретный запроса. практически это тот сервер, с который пользователь начать работа с t-libra в один из библиотек, включить в распределённый среду. 1) клиентский часть t-libra (функционировать в браузер пользователя), работать с какой-либо сервер t-libra («базовый» сервер), получать от базовый сервер список адрес другой серверов, который быть участвовать в полнотекстовый запрос (этота список вестись администратор базовый сервер t-libra). 2) после заполнение форма полнотекстовый запрос пользователь инициировать выполнение запрос (кнопка «выполнить запрос»). 3) браузер рассылать http-запрос (на выполнение полнотекстовый запроса) на базовый сервер и прочие, перечисленный в списке. 4) каждый сервер выполнять запрос, формировать результат и отсылать в клиентский часть 10 хороший ответ и общий размер результата. 5) из получить частичный «топ 10» (от каждый сервера) клиентский часть гарантированно строить и отображать пользователь «топ 10» ответ в целое (первый страница результата) и суммарный размер результата. 6) в ожидание реакция пользователь клиентский часть рассылать http-запрос (на пересылка остаток результата) на всё серверы, участвовать в выполнение запроса. 7) клиентский часть принимать ответ от них, объединять частичный результат и позволять пользователь просмотр следующий страница результат распределённый полнотекстовый запроса. этот уже реализовать алгоритм быть совершенствоваться по мера развитие и масштабирование проекта. 3.3. проект «humanitariana» (краткий описание) на
